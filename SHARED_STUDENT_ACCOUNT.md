# Изменения в системе аутентификации студентов

## Дата: 2 января 2026

### Основные изменения

#### 1. Общий студенческий аккаунт
Все студенты теперь используют один общий аккаунт для входа в систему:
- **Логин:** `student`
- **Пароль:** `123456`

#### 2. Множественные одновременные сессии
Система теперь позволяет нескольким студентам одновременно работать с одним логином. Каждый студент может:
- Войти в систему независимо от других
- Работать параллельно с другими студентами
- Не будет выкидывать других пользователей при входе

#### 3. Выбор взвода в расписании
В интерфейсе расписания студентов добавлен селектор взвода:
- Студенты могут выбрать свой взвод из выпадающего списка
- По умолчанию показываются все взводы
- После выбора взвода отображается только расписание выбранной группы

#### 4. Отключение создания студентов в админ-панели
- Создание студентов через админ-панель отключено
- Студенты больше не создаются индивидуально
- В форме создания пользователя убрана опция "Студент"
- API возвращает ошибку при попытке создать студента

### Технические детали

#### Backend изменения:
1. **auth/router.py**: Разрешены множественные сессии для студентов
2. **admin/router.py**: Заблокировано создание студентов через API
3. **timetable/router.py**: Добавлен параметр `classId` для фильтрации расписания

#### Frontend изменения:
1. **AdminUsersPage.tsx**: 
   - Убрана опция создания студентов
   - Добавлено информационное сообщение об общем аккаунте
2. **StudentTimetablePage.tsx**: 
   - Добавлен селектор взвода
   - Загрузка списка классов
   - Фильтрация расписания по выбранному взводу
3. **client.ts**: Обновлен API клиент для поддержки параметра `classId`

#### База данных:
- Миграция `20260102_000001_shared_student_account.sql` создает общий студенческий аккаунт

### Инструкции для администраторов

#### Применение изменений:
1. Применить миграцию базы данных:
   ```bash
   # Применить через Supabase или напрямую в PostgreSQL
   psql -f supabase/migrations/20260102_000001_shared_student_account.sql
   ```

2. Перезапустить API сервер:
   ```bash
   cd apps/api
   python -m uvicorn app.main:app --reload
   ```

3. Пересобрать фронтенд:
   ```bash
   cd apps/web
   npm run build
   ```

#### Информирование студентов:
Необходимо проинформировать всех студентов о новых учетных данных:
- Логин: **student**
- Пароль: **123456**
- При входе выбирать свой взвод в разделе "Расписание"

#### Управление взводами:
- Взводы (группы) создаются и управляются через раздел "Группы" в админ-панели
- Каждый студент должен знать название своего взвода
- Расписание создается для взводов через раздел "Расписание"

### Безопасность

**Важно:** Общий пароль `123456` является простым и предназначен для удобства студентов. Рекомендации:
- Рассмотреть возможность периодической смены пароля
- Убедиться, что система доступна только из доверенной сети
- Следить за журналами доступа к системе

### Возврат к старой системе

Если необходимо вернуться к индивидуальным студенческим аккаунтам:
1. Откатить изменения в `auth/router.py` (убрать логику множественных сессий)
2. Откатить изменения в `admin/router.py` (разрешить создание студентов)
3. Откатить изменения в frontend компонентах
4. Удалить общий студенческий аккаунт из базы данных

### Поддержка

При возникновении проблем проверьте:
- Логи API сервера
- Консоль браузера для ошибок фронтенда
- Правильность применения миграций базы данных
