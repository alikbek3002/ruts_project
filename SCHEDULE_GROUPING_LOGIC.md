# Логика группировки классов в расписании

## Как работает группировка

### Основной принцип
На **одной паре могут сидеть от 2 до 4 классов** одновременно (взводы).

### Примеры

#### Пример 1: 2 класса в потоке
```
Группа 1: [Класс А, Класс Б]
```
- Создается 1 запись расписания на каждое занятие
- Оба класса сидят вместе на всех парах
- Один преподаватель, одна аудитория

#### Пример 2: 4 класса в потоке
```
Группа 1: [Класс А, Класс Б, Класс В, Класс Г]
```
- Создается 1 запись расписания на каждое занятие
- Все 4 класса сидят вместе на всех парах
- Один преподаватель, одна аудитория

#### Пример 3: 5 классов в потоке
```
Группа 1: [Класс А, Класс Б, Класс В]
Группа 2: [Класс Г, Класс Д]
```
- Создается 2 записи расписания на каждое занятие
- Группа 1 (3 класса) сидит вместе в одно время
- Группа 2 (2 класса) сидит вместе в другое время
- Разные преподаватели/аудитории или в разные слоты

#### Пример 4: 7 классов в потоке
```
Группа 1: [Класс А, Класс Б, Класс В, Класс Г]
Группа 2: [Класс Д, Класс Е, Класс Ж]
```
- Создается 2 записи расписания на каждое занятие
- Группа 1 (4 класса) вместе
- Группа 2 (3 класса) вместе

## Алгоритм генерации

### Шаг 1: Группировка классов
```python
classes = [А, Б, В, Г, Д]
↓
groups = [
    [А, Б, В],  # Группа 1: 3 класса
    [Г, Д]      # Группа 2: 2 класса
]
```

### Шаг 2: Создание базового расписания
```python
Предмет: Математика
↓
Ищем доступный слот для ВСЕХ классов (А,Б,В,Г,Д)
↓
Находим: Понедельник 09:00-10:20
```

### Шаг 3: Создание записей для каждой группы
```python
Запись 1:
- Группа: [А, Б, В]
- Предмет: Математика
- Время: Понедельник 09:00-10:20
- Преподаватель: Иванов И.И.
- Аудитория: 201

Запись 2:
- Группа: [Г, Д]
- Предмет: Математика
- Время: Понедельник 09:00-10:20
- Преподаватель: Иванов И.И. (тот же)
- Аудитория: 202 (другая)
```

## Важно!

✅ **Все классы в потоке получают одинаковый набор занятий**
- Каждый предмет
- Одинаковое количество часов
- В одно и то же время

✅ **Преподаватель может вести несколько групп одновременно**
- Если групп 2-3, преподаватель может переходить между аудиториями
- Или это могут быть разные преподаватели на разные группы

✅ **Оптимизация использования ресурсов**
- Меньше записей в расписании
- Эффективное использование преподавателей
- Возможность проводить общие лекции

## Настройки

В файле `apps/api/app/modules/streams/scheduler.py`:

```python
MIN_CLASSES_PER_LESSON = 2  # Минимум классов в группе
MAX_CLASSES_PER_LESSON = 4  # Максимум классов в группе
```

Можно изменить эти константы, если нужно другое количество.

## Преимущества такого подхода

1. **Экономия преподавателей** - один преподаватель ведет несколько классов
2. **Гибкость** - можно проводить общие лекции для потока
3. **Оптимизация** - меньше конфликтов в расписании
4. **Справедливость** - все классы получают одинаковое расписание

## Как проверить результат

После генерации расписания:

1. Откройте расписание любого класса из потока
2. Проверьте поле `class_ids` в записях
3. Вы увидите массив классов, которые присутствуют на паре
4. Например: `["class-id-1", "class-id-2"]` = 2 класса вместе

## Пример вывода после генерации

```
Расписание успешно создано: 
24 занятий × 2 групп = 48 записей 
(по 2-2 классов в группе)

Журнал заполнен на 3 месяца (1440 записей).
```

Это означает:
- 24 уникальных занятия (предметы × время)
- 2 группы классов
- 48 записей в таблице расписания (24 × 2)
- Каждая группа содержит 2 класса
